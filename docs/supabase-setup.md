# Настройка Supabase (Этап 1)

Пошагово, для того кто делает это впервые. Всё бесплатно.

## 1. Создать проект

1. Открыть https://supabase.com → **Start your project** → войти через GitHub или email.
2. **New project**:
   - **Name:** `uchet-tovara` (любое)
   - **Database Password:** придумать надёжный пароль → **сохранить его** (пригодится для бэкапов).
   - **Region:** ближайший (например, Frankfurt / Central EU).
   - **Plan:** Free.
3. Подождать ~2 минуты, пока проект создастся.

## 2. Применить миграции (создать таблицы)

В проекте слева: **SQL Editor** → **New query**. Выполнить по очереди,
каждый файл отдельно, нажимая **Run**:

1. Содержимое `supabase/migrations/0001_init.sql`
2. Содержимое `supabase/migrations/0002_rls.sql`
3. Содержимое `supabase/migrations/0003_movement_trigger.sql`

Порядок важен. После каждого запроса должно быть «Success. No rows returned».

Проверить, что таблицы созданы: слева **Table Editor** → должны появиться
`boutiques`, `profiles`, `products`, `movements`.

## 3. Скопировать ключи в приложение

Слева: **Project Settings → API**. Скопировать:

- **Project URL** → в `.env.local` как `NEXT_PUBLIC_SUPABASE_URL`
- **anon public** ключ → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- **service_role** ключ → `SUPABASE_SERVICE_ROLE_KEY` (секретный! только сервер)

Создать файл `.env.local` в корне проекта (по образцу `.env.example`) и вставить значения.

## 4. Как проверяется изоляция бутиков

Важно: **SQL Editor работает от имени администратора БД и обходит RLS** —
поэтому изоляцию нельзя проверить прямыми запросами в редакторе.
Реальная проверка — через вход в приложение под двумя разными бутиками
(Этап 2). Тогда каждый пользователь увидит только свои товары.
