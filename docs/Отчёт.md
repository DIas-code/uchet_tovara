# Отчёт по проделанной работе — «Учёт товара»

**Дата:** 5 июля 2026
**Адрес приложения:** https://uchet-tovara-e1u1.vercel.app
**Исходный код:** https://github.com/DIas-code/uchet_tovara

---

## 1. Что это за приложение

Веб-приложение для учёта одежды в бутиках. Поддерживает **несколько бутиков**
одновременно: у каждого свой вход, свои товары и категории, и данные бутиков
**полностью изолированы** — один бутик физически не видит и не может изменить
данные другого (защита на уровне базы данных).

Интерфейс на русском, с заделом под казахский и английский.

## 2. Технологии и размещение

| Что | Чем |
|-----|-----|
| Приложение | Next.js 16 (React, TypeScript) |
| База данных + вход | Supabase (PostgreSQL + Auth + Row Level Security) |
| Размещение | Vercel (приложение) + Supabase (БД) — **бесплатно** |
| Код и авто-деплой | GitHub: при каждом изменении сайт пересобирается автоматически |

## 3. Разделы сайта и что они делают

### Вход (`/login`)
Вход по **логину и паролю** (без email). Логин привязан к конкретному бутику —
после входа человек видит только свой бутик.

> Технически Supabase логинит по email, поэтому логин внутри превращается в
> служебный адрес `логин@uchet.local` — пользователь этого не видит.

### Главная (`/`)
Плитки со статистикой бутика: сколько товаров, сколько единиц на складе,
общая стоимость склада (в ₸), сколько позиций закончилось.

### Товары (`/products`)
- Список товаров: код, наименование, категория, остаток, цена продажи.
- Поиск по коду/названию, фильтр «только в наличии».
- **Плашки-фильтры по категориям**, в режиме «Все» — группировка по категориям.
- Добавление, редактирование, удаление товара (удалять — только при нулевом остатке).
- Бутик к товару подставляется автоматически.

### Карточка товара (`/products/<код>`) — контроль количества
Открывается кликом по коду товара в списке. Здесь:
- текущий остаток и цена;
- кнопка **«Провести операцию»** — **Приход / Продажа / Возврат** с количеством и датой;
- **история всех операций** по товару.

Остаток пересчитывается автоматически: приход и возврат увеличивают, продажа
уменьшает. Продать больше, чем есть на складе, система **не даёт** (остаток не
уходит в минус).

### Категории (`/categories`)
Управление категориями бутика: добавить, переименовать, удалить.
У каждого бутика **свои** категории (название уникально в пределах бутика).
При удалении категории её товары становятся «без категории».

### Отчёты (`/reports`)
- Фильтр по периоду (с… по…).
- Сводка за период: продано (шт.), выручка (₸), приход, возврат.
- Таблица операций за период.
- **Экспорт в CSV**: выгрузка товаров и выгрузка операций (открывается в Excel).
- **Импорт из CSV**: массовая загрузка товаров файлом (кнопка «Импорт» на
  странице «Товары», только администратор) — новые категории создаются сами,
  остаток заводится приходом.

## 4. Роли пользователей

- **Администратор** — всё внутри своего бутика: товары, цены, удаление,
  категории, отчёты.
- **Продавец** — приход/продажа/возврат, просмотр; не может удалять товары,
  менять цены и управлять категориями.

## 6. Надёжность и данные

- **Изоляция бутиков** — на уровне PostgreSQL (Row Level Security).
- **Целостность остатков** — пересчёт в транзакции + запрет отрицательного остатка.
- **Keep-alive** — приложение раз в сутки пингуется, чтобы бесплатная база
  не засыпала.
- **Ежедневный бэкап данных** — через GitHub Actions (нужно один раз добавить
  секрет, см. `docs/backups.md`).
- Схема базы хранится в `supabase/migrations/` — воспроизводима в любой момент.
