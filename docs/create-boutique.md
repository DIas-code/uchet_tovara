# Создание бутика и пользователей (Этап 2)

Открытой регистрации нет — бутики и логины заводит владелец вручную.
Так безопаснее: чужой не создаст себе доступ.

## Создать бутик с администратором

### 1. Создать пользователя (логин/пароль)

Вход в приложение — по логину (без email). Supabase логинит по email, поэтому
логин заводится как служебный адрес **`<логин>@uchet.local`**.

В Supabase: **Authentication → Users → Add user → Create new user**.
- **Email:** `<логин>@uchet.local` (например, для логина `shop2` → `shop2@uchet.local`).
- **Password:** пароль (минимум 6 символов).
- Отметить **Auto Confirm User**.
- Нажать **Create user** и скопировать **User UID** (длинный id).

> Пользователь при входе вводит только логин (`shop2`) и пароль — адрес
> `@uchet.local` подставляется приложением автоматически.

### 2. Создать бутик и привязать пользователя

В **SQL Editor** выполнить (подставив своё имя бутика):

```sql
insert into public.boutiques (name)
values ('Бутик №1')
returning id;
```

Скопировать полученный `id` бутика. Затем:

```sql
insert into public.profiles (id, boutique_id, role, full_name)
values (
  '<USER_UID>',        -- UID из шага 1
  '<BOUTIQUE_ID>',     -- id из предыдущего запроса
  'admin',             -- роль: admin (полный доступ) или seller
  'Имя владельца'
);
```

Готово — этот пользователь может войти и видит только свой бутик.

## Добавить продавца в существующий бутик

1. Создать ещё одного пользователя (шаг 1 выше).
2. Привязать к тому же `boutique_id`, но с ролью `seller`:

```sql
insert into public.profiles (id, boutique_id, role, full_name)
values ('<USER_UID>', '<BOUTIQUE_ID>', 'seller', 'Имя продавца');
```

## Проверка изоляции (после входа в приложение)

1. Создать **два** бутика, в каждом по пользователю.
2. Войти под первым → добавить товар.
3. Выйти, войти под вторым → товаров первого не видно. Это и есть изоляция.

> Совет: роли `admin` / `seller` пока различаются в интерфейсе (Этап 3):
> продавец не сможет удалять товары и менять цены.
